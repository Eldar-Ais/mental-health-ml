# Анализ факторов психического здоровья сотрудников IT

Учебный проект по машинному обучению.  
Цель — проанализировать факторы, связанные с обращением сотрудников за лечением психического здоровья, и построить модель классификации уровня риска.

Кроме построения моделей, проект показывает влияние рабочих условий, личной истории и доступности помощи на вероятность обращения за лечением. Анализ объединяет EDA, работу с категориальными признаками и оценку нескольких моделей машинного обучения.

## Описание проекта

В проекте используется открытый датасет [Mental Health in Tech Survey (Kaggle)](https://www.kaggle.com/datasets/osmi/mental-health-in-tech-survey).  
Каждая строка — анонимный ответ сотрудника IT-сферы о его психическом состоянии, рабочей среде и доступности поддержки.

### Почему выбран именно этот датасет?

- содержит **личные и рабочие факторы**, влияющие на психическое здоровье;  
- много **категориальных признаков**, что делает задачу интересной для ML;  
- реальный социальный кейс, где важна интерпретируемость;  
- хорошо подходит для демонстрации EDA, кодирования признаков, работы с модельными метриками.

---

Основные шаги:

### 1. Предобработка данных

- фильтрация возраста (18–65 лет);  
- нормализация пола (`Gender_clean`);  
- преобразование целевой переменной `treatment` из `Yes/No` → `0/1`;  
- заполнение пропусков в категориальных признаках значением `"Unknown"`;  
- one-hot encoding категориальных данных;  
- масштабирование числовых признаков (`StandardScaler`).

**Зачем это делается?**

- чтобы удалить выбросы и некорректные значения (например, возраст 100+);  
- чтобы модели корректно работали с категориальными переменными;  
- чтобы избежать ошибок, связанных с пропусками;  
- чтобы SVM и Logistic Regression были устойчивее.

### 2. Разведочный анализ данных (EDA)

- анализ распределения признаков;  
- сравнение групп риска по возрасту и семейной истории;  
- визуализация с помощью гистограмм, boxplot, barplot;  
- построение ROC-кривых и корреляционной матрицы.

**Ключевые выводы EDA:**

- риск обращения выше у сотрудников старше **30 лет**;  
- наличие **семейной истории** психических заболеваний повышает вероятность обращения;  
- один из главных факторов — `work_interfere` (насколько работа мешает психическому здоровью);  
- влияние факторов **нелинейное**, поэтому деревья решений работают лучше линейных моделей.

### 3. Модели машинного обучения

В проекте используются три модели, каждая — с разной ролью:

- **Logistic Regression** — базовая интерпретируемая модель;  
- **Decision Tree** — основная модель, лучше всего работает на этих данных;  
- **SVM (RBF kernel)** — модель для проверки сложных нелинейных границ.

#### Зачем именно эти модели?

- Logistic Regression — чтобы понять **линейные связи** и влияние отдельных факторов.  
- Decision Tree — чтобы увидеть **правила принятия решений** и нелинейные комбинации признаков.  
- SVM — чтобы проверить, даёт ли сложная граница разделения прирост качества.

### 4. Оценка качества

Используются метрики:

- **Accuracy** — доля правильных ответов;  
- **Precision** — насколько точны предсказания “высокого риска”;  
- **Recall** — сколько реальных случаев риска модель нашла (особенно важно в задачах здоровья);  
- **F1-score** — баланс Precision и Recall;  
- **ROC AUC** и ROC-кривые — для анализа качества разделения классов по вероятности.

В задачах психического здоровья **важнее Recall и F1**, чем просто Accuracy, так как критично не пропускать людей из группы риска.
